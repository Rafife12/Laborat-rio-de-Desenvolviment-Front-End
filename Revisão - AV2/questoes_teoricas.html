Respostas Teóricas Frontend - Questões 1 a 18

1. Elementos Semânticos vs Não Semânticos

Semânticos: possuem significado claro sobre o conteúdo, facilitam SEO e acessibilidade.

<article>, <header>, <footer>, <nav>, <section>

Não semânticos: são elementos genéricos usados para estrutura, sem significado.

<div>, <span>

2. Correção do Código HTML

<!DOCTYPE html>
<html>
<head>
  <title>Página</title>
</head>
<body>
  <h1>Bem-vindo</h1>
  <p>Conteúdo da página</p>
</body>
</html>

Justificativa: Fechamento correto das tags, fechamento do <head> e uso correto das tags de título e parágrafo.

3. Uso do atributo alt na tag <img>

O atributo alt fornece descrição alternativa para a imagem, essencial para acessibilidade e SEO.

<img src="foto.jpg" alt="Descrição da imagem" />

4. Formulário com método POST

<form method="POST" action="/enviar">
  <label>Nome:</label>
  <input type="text" name="nome" required />
  <label>Email:</label>
  <input type="email" name="email" required />
  <button type="submit">Enviar</button>
</form>

5. DOCTYPE em HTML5

A declaração <!DOCTYPE html> indica ao navegador que o documento é HTML5, ativando o modo padrão.

6. Diferença entre classes e IDs no CSS

/* Classe - pode ser usada em vários elementos */
.elemento {
  color: blue;
}

/* ID - único na página */
#elementoUnico {
  font-weight: bold;
}

7. Correção do CSS

#titulo {
  color: red;
  font-size: 20px;
  text-align: center;
}

8. Modelo de Caixa (Box Model)

Composto por: conteúdo, padding (preenchimento interno), border (borda) e margin (margem externa).

/* Exemplo box model */
div {
  width: 200px;       /* conteúdo */
  padding: 10px;      /* espaço interno */
  border: 2px solid;  /* borda */
  margin: 15px;       /* margem */
}

9. Estilo CSS para parágrafos

p {
  font-style: italic;
  color: blue;
  margin-top: 20px;
}

10. Seletores CSS

/* Seletor por tag */
p {
  color: green;
}

/* Seletor por classe */
.minha-classe {
  font-weight: bold;
}

/* Seletor por ID */
#meu-id {
  font-size: 18px;
}

11. Variáveis em JavaScript: var, let e const

// var - escopo global ou de função, redeclarável
var x = 10;

// let - escopo de bloco, não redeclarável no mesmo bloco
let y = 20;

// const - constante, não pode ser reatribuída
const z = 30;

12. Correção da função JavaScript

function exibirMensagem() {
  let nome = prompt("Digite seu nome");
  alert("Olá " + nome);
}

13. Funções anônimas e Arrow Functions

// Função anônima
const anonima = function() {
  console.log("Função anônima");
};

// Arrow function
const seta = () => {
  console.log("Arrow function");
};

<button id="btn">Clique aqui</button>

document.getElementById('btn').addEventListener('click', () => {

15. Uso do querySelector()

document.querySelector("#titulo").textContent = "Novo conteúdo";

16. Estruturas de controle if, else if e switch

let x = 2;

if (x === 1) {
  console.log("Um");
} else if (x === 2) {
  console.log("Dois");
} else {
  console.log("Outro");
}

switch(x) {
  case 1:
    console.log("Um");
    break;
  case 2:
    console.log("Dois");
    break;
  default:
    console.log("Outro");
}

17. API REST e métodos HTTP

API REST usa HTTP para comunicação. Métodos principais:

  • GET: Recuperar dados
  • POST: Criar dados
  • PUT: Atualizar dados
  • DELETE: Apagar dados

18. Diferença entre GET e POST

GET: Envia parâmetros na URL, usado para buscar dados, seguro e idempotente.

POST: Envia dados no corpo da requisição, usado para criar/alterar dados, não idempotente.

// Exemplo GET
GET /usuarios?id=123

// Exemplo POST
POST /usuarios
Content-Type: application/json

{
  "nome": "João"
}
